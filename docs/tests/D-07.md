## 1. 功能範圍

提供使用者 / 系統可定時觸發的提醒物件，支援：

| 能力                                                  | 狀態 |
| ----------------------------------------------------- | ---- |
| 單次提醒 (`recurrence: 'none'`)                       | ✅   |
| 週期：daily / weekly / monthly / yearly + interval    | ✅   |
| 綁定 target：database_row / task（驗 type↔id 成對）   | ✅   |
| lifecycle：active / paused / completed + soft-delete  | ✅   |
| `advance()` — 排程器觸發後推進 `nextFireAt`           | ✅   |
| 月底 / 閏年 clamp（Jan 31 + 1M → Feb 28/29）         | ✅   |
| 實際 cron / scheduler 輪詢發送                        | ❌ 下個循環    |
| 推播通知 (FCM/APNs) 對接 C-14                         | ❌ (C-14)     |
| 跨 workspace 的聚合檢視                               | ❌            |

## 2. Data 模型

```prisma
model Reminder {
  id                 String    @id @default(uuid())
  workspaceId        String
  userId             String        // 建立者
  title              String
  notes              String?
  targetType         String?       // database_row | task
  targetId           String?
  triggerAt          DateTime      // 首次觸發
  recurrence         String    @default("none")
  recurrenceInterval Int       @default(1)
  nextFireAt         DateTime      // scheduler 輪詢欄
  lastFiredAt        DateTime?
  status             String    @default("active") // active | paused | completed
  createdAt          DateTime  @default(now())
  updatedAt          DateTime  @updatedAt
  deletedAt          DateTime?
}
```

Index：
- `(nextFireAt, status, deletedAt)` — 給未來 scheduler 做輪詢
- `(workspaceId, deletedAt)` — 列表查詢

## 8. 已知限制

1. **無排程器** — `advance()` 需外部觸發；建 Reminder 不會自動跑。待實作 NestJS `@Cron` 或 BullMQ 輪詢。
2. **Timezone** — 全部用 UTC，前端顯示須自行換算 workspace / user 時區。
3. **missed fires** — 若排程器 down 導致錯過，`advance` 只推進 1 interval，不會一次補齊。
4. **target referential integrity** — `targetType+targetId` 只存文字，不驗證實際資料列 / 任務是否存在。
5. **recurrence 改變** — 更新 `recurrence` 時，`nextFireAt` 會重設為 `triggerAt`（符合使用者重新調時的預期，但若想保留既有週期需另作 UI 流程）。
