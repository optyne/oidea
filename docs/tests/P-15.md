## 1. 功能範圍

條件觸發動作。**MVP 聚焦**：

| 類別     | MVP 支援                                                           |
| -------- | ------------------------------------------------------------------ |
| Trigger  | ✅ `task_completed`（Task 首次完成）                               |
|          | ❌ `task_moved_to_column` / `row_status_changed` / time-based     |
| Scope    | ✅ `project` / `database`（目前 engine 僅走 project）              |
| Action   | ✅ `notify_user`（`NotificationsService.create`）                  |
|          | ✅ `post_to_channel`（`MessagesService.create`、用 createdBy 身份）|
|          | ❌ `assign_user` / `move_column` / `create_reminder`              |
| 模板     | ✅ `{{task.title}}` / `{{task.id}}` / `{{task.dueDate}}`           |
| 失敗容錯 | ✅ 單一 rule 噴錯不阻塞其他 rule（logged only）                    |

BILLING_CONTRACTS 的 "合約任務完成 → 在 #財務 公告" 直接可用；
"狀態>7 天 → @財務" 屬時間型，留下一刀（可接 D-07 scheduler）。

## 2. Schema

```prisma
model AutomationRule {
  id            String    @id @default(uuid())
  workspaceId   String
  createdBy     String
  name          String
  description   String?
  scope         String    // project | database
  scopeId       String
  trigger       String    // task_completed | ...
  triggerConfig Json?
  action        String    // notify_user | post_to_channel
  actionConfig  Json
  enabled       Boolean   @default(true)
  ...timestamps + soft-delete
}
```

Index：
- `(scope, scopeId, trigger, enabled, deletedAt)` — engine 批次查
- `(workspaceId, deletedAt)` — 列表

---

## 3. API

```
POST   /automation-rules                         建立
GET    /automation-rules?workspaceId=…           列表
GET    /automation-rules/:id
PATCH  /automation-rules/:id                     更新（僅建立者）
DELETE /automation-rules/:id                     軟刪（僅建立者）
```

`actionConfig` 依 `action` 不同：
- `notify_user`: `{ userId, title?, contentTemplate? }`
- `post_to_channel`: `{ channelId, contentTemplate }`

---

## 4. Engine 合約

```ts
class AutomationEngine {
  onTaskCompleted(task: { id, projectId, title, assigneeId?, dueDate? }): Promise<number>
}
```

由 `TasksService.update()` 在「首次完成轉換」時呼叫；回傳成功 dispatch 的 rule 數。

## 10. 已知限制

1. **trigger 僅 task_completed** — 未來 column move、status change、time-based 需擴 engine 公用介面。
2. **post_to_channel 以 rule.createdBy 身份送** — 若建立者已退出頻道 → MessagesService 擋住、engine log 為失敗；規則不自動停用。
3. **actionConfig 型別 loose** — JSON 存放，驗證只在 create/update 時跑；日後改多型別 DTO。
4. **模板簡易 regex** — 不支援巢狀欄位、條件式、i18n；考慮 Handlebars 之後再換。
5. **無批次觸發 API** — Engine 一事件處理一次；沒 debounce。
6. **無 audit log** — 沒有 per-rule 的 fire 歷程；需另開表或接 Activity。
7. **權限以「建立者」為準** — 尚未支援 workspace admin 代管他人 rule。
