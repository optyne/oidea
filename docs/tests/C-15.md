## 1. 功能範圍

讓員工把常用公告、請款提醒、結算流程等文案存成可重用的 snippet；之後配合 C-16 廣播可一鍵發送到多個群組。

| 能力                                                       | 狀態 |
| ---------------------------------------------------------- | ---- |
| CRUD：建立 / 列表 / 讀單筆 / 更新 / 軟刪                   | ✅   |
| 可見度：`personal`（僅作者）/ `workspace`（同 ws 成員皆可用）| ✅   |
| 權限：僅作者可 update / remove（即使 visibility=workspace） | ✅   |
| `shortcut` 正規化驗證 (`/xxx`、僅英數與 dash)              | ✅   |
| 查詢排序：`updatedAt desc`                                 | ✅   |
| 與 C-16 廣播的自動帶入                                      | ❌ 下一刀串接 |
| 使用頻次 / 最後使用時間                                     | ❌   |
| 全 workspace 的 shortcut 去重                               | ❌（目前靠前端避重）|

## 2. Data 模型

```prisma
model MessageSnippet {
  id          String    @id @default(uuid())
  workspaceId String
  createdBy   String
  name        String
  content     String
  shortcut    String?                     // /billing-notice
  visibility  String    @default("personal")
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt
  deletedAt   DateTime?
}
```

Index：
- `(workspaceId, deletedAt, visibility)` — 列表常用條件
- `(createdBy, deletedAt)` — 使用者自己的 personal snippet

---

## 3. API

```
POST   /snippets                { workspaceId, name, content, shortcut?, visibility? }
GET    /snippets?workspaceId=…   → (workspace-visible + 自己的 personal)
GET    /snippets/:id
PATCH  /snippets/:id             { name?, content?, shortcut?, visibility? }
DELETE /snippets/:id             (軟刪；僅作者)
```

錯誤碼：
- 400 `shortcut` 格式錯 / name 或 content 空 / visibility 白名單外
- 403 非成員 / 非作者 / 讀他人 personal
- 404 不存在或已軟刪

## 9. 已知限制

1. **shortcut 不 unique** — 允許重複，用者自行負責；未來要做 auto-complete 時會加 `(workspaceId, shortcut)` unique 限制。
2. **沒有分類 / 標籤** — MVP 只有一層清單，量大時需加 folders / tags。
3. **沒有版本歷史** — update 直接覆蓋；對合規要求高時未來補 `SnippetRevision`。
4. **personal 可見性後移** — 若改 visibility 由 workspace → personal，原本共用者將看不到（符合預期，但需提醒）。
5. **admin 改不了他人 snippet** — 目前只以「作者」為權限單位；若要給 workspace admin 管理權，日後加 role 判斷。
