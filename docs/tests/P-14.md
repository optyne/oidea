## 1. 功能範圍

每年 / 每月 / 每週 / 每日循環任務（例：年度維護費請款）。實作策略：**事件驅動** —
任務被標記完成（`completedAt` 首次設值）時，立即根據 `recurrence` + `recurrenceInterval` + `dueDate` 生成下一張任務卡。不用另設 cron。

| 能力                                                         | 狀態 |
| ------------------------------------------------------------ | ---- |
| create 指定 `recurrence` / `recurrenceInterval`              | ✅   |
| 首次完成時自動 spawn 下一張 (相同 title / priority / assignee) | ✅   |
| `recurringSourceId` 指向序列根 (第二代以後沿用不覆蓋)        | ✅   |
| `startDate` 相對 `dueDate` 的偏移會跟著前進                  | ✅   |
| 月底 / 閏年 clamp（沿用 `addRecurrence`）                    | ✅   |
| 重複送 `completed=true` 不重複 spawn                         | ✅   |
| `completed=false`（取消完成）不 spawn                        | ✅   |
| Recurrence 同步到整個序列（「這張之後全部」modify）          | ❌ MVP 僅改本張 |
| 循環結束條件（例：跑 N 次後停）                              | ❌   |
| 暫停 / 恢復循環                                              | ❌ (改 recurrence=none 自然停) |

## 2. Schema 變更

`Task` 加：
```
recurrence         String  @default("none")   // none | daily | weekly | monthly | yearly
recurrenceInterval Int     @default(1)
recurringSourceId  String?
recurringSource    Task?   @relation("TaskRecurringSeries", onDelete: SetNull)
recurringInstances Task[]  @relation("TaskRecurringSeries")
@@index([recurringSourceId])
```

> `onDelete: SetNull`：刪掉根任務後，序列實例仍可存活但失去血緣。

## 3. 流程

```
PATCH /tasks/:id  { completed: true }
 ↓
TasksService.update()
 ↓   (偵測首次完成 + recurrence !== 'none' + dueDate 存在)
spawnRecurringInstance()
 ↓
create new Task with:
  - nextDue   = addRecurrence(old.dueDate, rule, interval)
  - nextStart = nextDue + (old.startDate - old.dueDate)  [如有]
  - recurringSourceId = old.recurringSourceId ?? old.id
  - completedAt = null, 其他欄位繼承
```

## 9. 已知限制

1. **完成時才 spawn** — 若使用者忘記標完成，不會自動生成；想看到未來任務只能手動建或等完成觸發。
2. **無「跑完就停」** — 除非手動改 recurrence='none'，否則無限延伸。
3. **title 改不會回填** — 改了本張 title 不影響先前 / 未來 spawn 的實例（因為下張是完成時從當下 snapshot）。
4. **Activity log 未記 spawn 事件** — 生下一張不會寫 TaskActivity，僅 update 本張會。
5. **沒有 workspace 權限檢查** — 沿用 tasks 現有實作（本 PR 不擴展既有限制）。
6. **UTC-based 推進** — dueDate 時區由前端處理。
