## 1. 功能範圍

把一則聊天訊息一鍵轉成 Task — 對應員工工作流「群裡討論完的事項要追就建一張卡」。

本刀只做 **message → task**；message → database_row、message → snippet 留下一刀。

| 能力                                                   | 狀態 |
| ------------------------------------------------------ | ---- |
| 任意頻道訊息 → Task（保留 `sourceMessageId` 回溯）    | ✅   |
| 預設 title / description / priority / 可覆蓋           | ✅   |
| 指定 assignee（驗證同 workspace 成員）                | ✅   |
| 專案 / 欄位 必須在同 workspace + 欄屬該專案           | ✅   |
| 從 Task 反向查「源自哪則訊息」                         | ✅（`@@index([sourceMessageId])`）|
| 訊息 → database_row / snippet                         | ❌ 下一刀 |
| 批次轉換（多訊息一次建多個任務）                       | ❌   |

---

## 2. Schema 變更

`Task` 加：
```
sourceMessageId String?     // FK → Message
sourceMessage   Message? @relation("MessageConvertedTasks", onDelete: SetNull)
@@index([sourceMessageId])
```

`Message` 加 back-ref：
```
convertedTasks Task[] @relation("MessageConvertedTasks")
```

> `onDelete: SetNull`：源訊息被刪 / 軟刪後，Task 仍存活但 source 變 null。

---

## 3. API

```
POST /messages/:id/convert-to-task
Authorization: Bearer <jwt>
{
  "projectId":  "p-1",
  "columnId":   "col-1",
  "title":      "自訂 (可省略，預設取訊息前 100 字)",
  "description":"自訂 (可省略，預設為完整訊息)",
  "priority":   "urgent | high | medium | low",
  "assigneeId": "u-2",
  "dueDate":    "2026-05-31"
}
→ 200 Task
```

錯誤碼：
- 400 專案他 workspace / 欄位不屬該專案 / assignee 非 ws 成員
- 403 非頻道成員
- 404 訊息 / 專案 / 欄位 不存在

## 9. 已知限制

1. **單筆轉換** — 沒有批次 API；選多則訊息要前端迴圈呼叫。
2. **無回寫訊息** — 轉成 Task 後，源訊息不會出現「已建立任務 #123」的連結；要等 C-19（內部連結預覽）串起來。
3. **assignee 的 ws 驗證不看 project role** — 只驗 WorkspaceMember；若 project 有獨立 ACL 未來要擴充。
4. **dueDate 僅接受 ISO 字串** — 客戶端須自行轉時區。
5. **history** — 沒寫 TaskActivity（既有 create 路徑也沒寫，屬全 module 一致限制）。
