## 1. 功能範圍

D-02 提供工作空間內可自訂欄位的結構化資料集（等同 Notion Database 精簡版），支援：

| 能力                                     | 本版支援 |
| ---------------------------------------- | -------- |
| 資料庫 CRUD (建立 / 列表 / 讀取 / 更新 / 軟刪) | ✅       |
| 欄位管理 (新增 / 更新 / 刪除)            | ✅       |
| 欄位型別：`text` / `number` / `date` / `select` | ✅       |
| 資料列 CRUD (含 cell upsert、軟刪)       | ✅       |
| 必填欄位驗證                             | ✅       |
| 型別驗證 (值必須符合欄位型別)             | ✅       |
| 多檢視 (Table / Kanban / Calendar)       | ❌ (D-03) |
| 檔案附件欄位                             | ❌ (D-04) |
| 公式欄位                                 | ❌ (D-06) |

## 3. 測試案例 (TC)

> 編號規則：`TC-{功能ID}-{三位流水號}`。001~009 = database CRUD、010~019 = column、020~029 = row add、030~039 = row update/remove。

### 3.1 Database CRUD

| ID            | 情境                   | 前置                  | 動作                | 預期                                                          |
| ------------- | ---------------------- | --------------------- | ------------------- | ------------------------------------------------------------- |
| TC-D02-001    | 成員建立資料庫         | user 為 ws 成員       | `create()`          | Prisma 寫入，`createdBy` = userId                            |
| TC-D02-002    | 非成員建立被拒絕       | 無 membership         | `create()`          | 丟 `ForbiddenException`，不呼叫 `database.create`            |
| TC-D02-003    | 讀取含欄位與有效列     | db + rows (含軟刪一筆) | `findById()`        | 回傳含 columns、rows；軟刪列被濾掉 (`where.deletedAt: null`) |
| TC-D02-004    | 不存在的 db            | —                     | `findById()`        | 丟 `NotFoundException`                                        |
| TC-D02-005    | 軟刪資料庫             | 已存在                | `remove()`          | `database.update` 寫入 `deletedAt`                            |

### 3.2 Column

| ID            | 情境                          | 前置                  | 動作                    | 預期                                       |
| ------------- | ----------------------------- | --------------------- | ----------------------- | ------------------------------------------ |
| TC-D02-010    | 新增 text 欄位                | 成員 + db 存在        | `addColumn(text)`       | 建立成功                                   |
| TC-D02-011    | select 沒給 choices           | 成員 + db 存在        | `addColumn(select, -)`  | `BadRequestException`                      |
| TC-D02-012    | select 帶 choices             | 成員 + db 存在        | `addColumn(select, 2 choices)` | 建立成功                              |
| TC-D02-013    | 不支援的 type (如 file)       | 成員 + db 存在        | `addColumn('file')`     | `BadRequestException`                      |

### 3.3 Row — Add

| ID            | 情境                          | 欄位配置                | 輸入                    | 預期                      |
| ------------- | ----------------------------- | ----------------------- | ----------------------- | ------------------------- |
| TC-D02-020    | 正常建立                      | text + number           | `{text: 'A', number: 60000}` | 建立成功                |
| TC-D02-021    | text 欄塞數字                 | text                    | `{text: 123}`           | `BadRequestException`     |
| TC-D02-022    | number 欄塞非數字             | number                  | `{number: 'abc'}`       | `BadRequestException`     |
| TC-D02-023    | date 欄需 ISO 字串            | date                    | `'不是日期'` / `'2026-04-20'` | 前者拒絕，後者通過    |
| TC-D02-024    | select 值不在 choices         | select (choices=[r])    | `{sel: 'x'}`            | `BadRequestException`     |
| TC-D02-025    | 必填欄位缺值                  | text required=true      | `{}`                    | `BadRequestException`     |
| TC-D02-026    | 引用不存在欄位                | (none)                  | `{ghost: 'x'}`          | `BadRequestException`     |

### 3.4 Row — Update / Remove

| ID            | 情境                          | 前置                      | 動作                     | 預期                                          |
| ------------- | ----------------------------- | ------------------------- | ------------------------ | --------------------------------------------- |
| TC-D02-030    | upsert cell                   | row 存在                  | `updateRow({text: 'B'})` | `databaseCell.upsert` 以 rowId+columnId 為鍵 |
| TC-D02-031    | 已軟刪的 row                  | row.deletedAt 有值        | `updateRow()`            | `NotFoundException`                           |
| TC-D02-032    | 軟刪 row                      | row 存在                  | `removeRow()`            | `databaseRow.update` 寫入 `deletedAt`         |

## 7. 未測到 / 已知限制

1. 並發更新同一 row 的不同 cell 無 optimistic locking — 最後寫入者勝。
2. Column 改型別（例如 text → number）會造成既有 cell 值型別不符，目前未阻擋 — **暫不支援改型別**，只能砍重建。
3. `select` 的 `choice.id` 沒做成 unique 限制，重複 id 的資料由前端避免。
4. Row position 的重排未有批次 API，需前端逐筆 update。
5. Integration / E2E 測試未建 — 下次進度要補。
