## 1. 功能範圍

指定時間把訊息送到一個 / 多個頻道。實作方式：到時間由 cron 呼叫 C-16 的 `broadcast()`，自動復用同一套驗證與 WS emit。

對應員工工作流：**每月 25 號自動發結算公告到財務 + PM 群組**。
組合 C-15 範本 → C-17 排程 → C-16 廣播，可完全自動化。

| 能力                                                   | 狀態 |
| ------------------------------------------------------ | ---- |
| 建立排程（1-50 channels、未來時間）                    | ✅   |
| 建立時 snapshot content / type / metadata              | ✅   |
| 列表（預設只看 pending，可加 `includeHistory=true`）   | ✅   |
| 取消（僅建立者、僅 pending）                           | ✅   |
| 軟刪（僅建立者）                                       | ✅   |
| `@Cron(EVERY_MINUTE)` 到期觸發                         | ✅   |
| fire 時交給 C-16 broadcast → 沿用成員驗證 + 原子寫入   | ✅   |
| 失敗寫 `failedReason`（fire 時不是成員之類）           | ✅   |
| 循環排程（月月結算）                                   | ❌ 可用 D-07 recurrence + auto-recreate；日後再做 |
| 編輯已建立排程                                         | ❌ 目前 cancel + 重建                              |

## 2. Data 模型

```prisma
model ScheduledMessage {
  id              String    @id @default(uuid())
  workspaceId     String
  createdBy       String       // 發送人；fire 時會以此身份呼 broadcast
  channelIds      String[]
  content         String?
  type            String    @default("text")
  metadata        Json?
  sendAt          DateTime
  status          String    @default("pending") // pending | sent | canceled | failed
  sentAt          DateTime?
  sentBroadcastId String?                       // C-16 broadcast() 回傳值
  failedReason    String?
  createdAt       DateTime  @default(now())
  updatedAt       DateTime  @updatedAt
  deletedAt       DateTime?
}
```

Index：
- `(sendAt, status, deletedAt)` — scheduler 輪詢
- `(workspaceId, deletedAt)` — 列表查詢

## 3. API

```
POST   /scheduled-messages                        { workspaceId, channelIds[], sendAt, content?, type?, metadata? }
GET    /scheduled-messages?workspaceId=…(&includeHistory=true)
GET    /scheduled-messages/:id
POST   /scheduled-messages/:id/cancel
DELETE /scheduled-messages/:id                    (軟刪)
```

錯誤碼：
- 400 sendAt 過去 / 頻道跨 workspace / 頻道空
- 403 非 workspace 成員、非頻道成員、cancel 非本人
- 404 頻道 / 排程不存在

| ID         | 情境                     | 預期                                               |
| ---------- | ------------------------ | -------------------------------------------------- |
| TC-C17-001 | member + 正常未來時間   | 建立 pending                                        |
| TC-C17-002 | sendAt 在過去            | `BadRequestException`                              |
| TC-C17-003 | 非 workspace 成員        | `ForbiddenException`                               |
| TC-C17-004 | 頻道在他 workspace       | `BadRequestException`                              |
| TC-C17-005 | 非頻道成員               | `ForbiddenException`                               |
| TC-C17-006 | 頻道不存在               | `NotFoundException`                                |
| TC-C17-007 | channelIds 重複           | 去重後寫入                                          |

### 5.2 Service — Query (TC-C17-010 ~ 011)

| ID         | 情境                         | 預期                                               |
| ---------- | ---------------------------- | -------------------------------------------------- |
| TC-C17-010 | 預設 list                    | where 有 `status: 'pending'`                       |
| TC-C17-011 | `includeHistory=true`        | 不過濾 status                                       |

### 5.3 Service — Cancel (TC-C17-020 ~ 022)

| ID         | 情境                       | 預期                           |
| ---------- | -------------------------- | ------------------------------ |
| TC-C17-020 | 建立者取消 pending         | `status = 'canceled'`           |
| TC-C17-021 | 非建立者                   | `ForbiddenException`           |
| TC-C17-022 | 已 sent                    | `BadRequestException`          |

### 5.4 Scheduler — pollAndFire (TC-C17-SC01 ~ SC06)

| ID          | 情境                        | 預期                                                   |
| ----------- | --------------------------- | ------------------------------------------------------ |
| TC-C17-SC01 | 到期 → broadcast 成功      | `markSent(id, now, broadcastId)`                       |
| TC-C17-SC02 | broadcast 噴錯             | `markFailed(id, reason)`                               |
| TC-C17-SC03 | 批次中一筆失敗             | 其他筆繼續完成；回傳 `{sent, failed}`                  |
| TC-C17-SC04 | 無到期                     | 零呼叫 broadcast / mark*                               |
| TC-C17-SC05 | content = null             | 傳 `undefined` 給 broadcast（不傳空字串）              |
| TC-C17-SC06 | markFailed 也失敗          | 只 log，不拋；`pollAndFire` 正常結束                   |

## 9. 已知限制

1. **UTC only** — `sendAt` 傳 ISO；前端需換算 user / workspace 時區。
2. **單節點 cron** — `@Cron` 在每個 Node 實例都會跑，多副本時需 advisory lock。
3. **無循環** — 月月結算要自己每次重建；之後可加 `recurrence` 欄位共用 D-07 helper。
4. **sender 退出頻道** — fire 時會被 `broadcast()` 擋；狀態變 failed，符合預期。
5. **內容 snapshot** — 建立後改 snippet 不影響；反之亦然。若想「連動 snippet」需另設連結型欄位。
6. **無編輯** — 想改內容就 cancel 後重建，避免 race condition。
7. **批次上限 200** — scheduler 一輪最多處理 200 筆，排程高峰期會延後到下一輪。
